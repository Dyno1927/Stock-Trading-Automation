# Code of Conduct

# Stock Trading Automation (STA)

Version: Draft

Status: Active

---

# Our Commitment

Stock Trading Automation (STA) is committed to fostering a welcoming, respectful, and professional environment for everyone who contributes to the project.

Whether contributing code, documentation, research, ideas, or feedback, every participant should feel comfortable collaborating within the community.

The project's success depends not only on technical excellence but also on constructive communication and mutual respect.

---

# Expected Behavior

All contributors are expected to:

* Be respectful.
* Be professional.
* Be patient.
* Be constructive.
* Be open to feedback.
* Support evidence-based technical discussions.
* Help maintain a welcoming environment.

Disagreements are normal during engineering discussions, but they should always remain respectful and focused on solving technical problems.

---

# Unacceptable Behavior

The following behaviors are not tolerated:

* Harassment
* Personal attacks
* Discrimination
* Hate speech
* Threats
* Trolling
* Deliberate misinformation
* Spam
* Publishing private information
* Malicious contributions

Technical criticism is encouraged.

Personal criticism is not.

---

# Engineering Discussions

Technical discussions should prioritize:

* Facts
* Benchmarks
* Documentation
* Industry standards
* Long-term maintainability

Avoid arguments based solely on personal preference when objective evidence is available.

---

# Decision Making

Engineering decisions should consider:

* Architecture
* Maintainability
* Security
* Performance
* Scalability
* Reliability

Consensus is preferred whenever practical.

When consensus cannot be reached, decisions should align with the documented architecture and long-term project vision.

---

# Reporting Issues

If you experience or witness unacceptable behavior:

* Remain respectful.
* Document the issue.
* Report it privately to the project maintainers.

Reports should include enough information to investigate the issue fairly.

---

# Enforcement

Project maintainers reserve the right to:

* Remove inappropriate content.
* Reject contributions.
* Limit participation.
* Suspend or permanently ban contributors for severe or repeated violations.

Enforcement actions should always be proportional to the severity of the issue.

---

# Scope

This Code of Conduct applies to:

* GitHub repositories
* Pull Requests
* Issues
* Discussions
* Documentation
* Project communication channels
* Community spaces associated with STA

---

# Attribution

This Code of Conduct is inspired by established open-source community standards while being adapted to the goals and culture of the Stock Trading Automation project.

---

# Guiding Principle

Build software professionally.

Treat people professionally.

Respect is one of the project's most important engineering practices.
