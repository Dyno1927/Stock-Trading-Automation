# Purpose

This document contains high-level release notes for each published version of Stock Trading Automation (STA).

Unlike `CHANGELOG.md`, which records every notable change, this document summarizes the most important milestones, architectural improvements, and major features introduced in each release.

---

# Versioning Strategy

STA follows Semantic Versioning (SemVer).

```text
MAJOR.MINOR.PATCH
```

### MAJOR

Breaking architectural or compatibility changes.

### MINOR

New functionality, major features, or completed project phases.

### PATCH

Bug fixes, documentation improvements, security fixes, and minor enhancements.

# Release Process

Before publishing a release, verify:

- All planned features are complete.
- Tests pass successfully.
- Documentation is updated.
- CHANGELOG.md has been updated.
- Architecture remains consistent.
- Security review is complete.
- Version numbers are updated.
- Release notes are prepared.

---

# Release Checklist

- Repository is clean
- Documentation is synchronized
- Tests are passing
- Version tag created
- CHANGELOG updated
- RELEASES updated
- GitHub Release prepared
- Release branch merged
- Release tag published
